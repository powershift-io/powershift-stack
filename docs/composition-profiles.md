# Composition-Aware Profiles

The PowerShift® Constitution is the first governance framework that adapts its own weight based on the human-to-agent ratio in the organization.

---

## The Problem with One-Size-Fits-All

Traditional governance frameworks (Holacracy, Sociocracy, even corporate bylaws) assume a fixed organizational composition: all human participants. This creates two problems as intelligent agents enter the picture:

1. **Dead weight.** Large sections of human-ergonomic governance scaffolding become inert when most participants are agents. Meeting rituals designed for emotional attunement, election processes for democratic selection, prioritization duties for human attention management — agents don't need these.

2. **Missing provisions.** The framework has nothing to say about capability envelopes, hard constraints, kill switches, or delegated sub-agents. You're bolting on amendments to a structure that didn't anticipate non-human participants.

## The Solution: Profiles

A **profile** is a composition-specific compilation of the Constitution that removes provisions inert for the target composition. Each profile is a *proper subset* of the one above it — **nothing added, only removed**. Every authority rule survives in every profile.

### Full Constitution (100%)

**For:** Any composition — many humans, many agents, or any mix.

Contains all provisions, including multi-party human process scaffolding (meeting rituals, elections, individual initiative, circle reps, surrogates).

### Agent Profile (~55%)

**For:** Few humans, many agents. AI-majority organizations.

**Removes:**
- Duty of Prioritization (agents prioritize algorithmically)
- Tactical meeting choreography (coordination is async/programmatic)
- Individual Initiative (agents operate within Capability Envelopes; improvisation outside authority is prevented by Hard Constraints)

**Preserves:** Election processes, circle reps, governance meetings, and all multi-party governance scaffolding (because some humans are still participating).

### Solo Profile (~30%)

**For:** One human Orchestrator + agent constellation.

**Additionally removes:**
- Circle Reps (unnecessary with one human overseeing all)
- Integrative Election Process (Orchestrator assigns directly)
- Surrogates (agents are continuously available)
- Governance meeting attendance/notice rules (governance is async-first)
- Linking provisions (advanced multi-team feature)

**Result:** Compact enough to serve as — or be embedded in — an agent's system prompt. Every authority rule intact. The Orchestrator can still run the full governance process; the scaffolding for *other humans* participating in that process is simply absent because there are no other humans.

## How to Choose

```
How many humans participate in governance?

  ├── Many (5+)          → Full Constitution
  ├── Few (2-4)          → Agent Profile
  └── One (solo founder) → Solo Profile
```

## Upgrading Profiles

Every profile includes an appendix cataloging every omission with full Constitution cross-references. If your composition changes — you admit a human partner, you grow the team — there's a clear path back to the fuller profile:

1. Identify which provisions become relevant for the new composition
2. Process a governance tension in the Anchor Circle to adopt the appropriate profile
3. The broader profile contains everything the narrower one had, plus the reinstated provisions

No migration. No rewrite. Just re-scope.

## The Key Invariant

**Every authority rule survives in every profile.** Who can do what, who controls which domains, how governance changes — all identical. The only difference is which *process scaffolding* is present. This is what makes profiles safe: you never lose a constraint by going narrower, and you never gain one by going broader.
